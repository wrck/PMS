# 知识库深度分析标准与完成情况

> 本文档定义了知识库深度分析的标准，并记录各模块的完成情况。

---

## 1. 深度分析标准

### 1.1 模块文档标准（02-modules）

每个模块文档必须包含：

1. **模块概述**
   - 模块职责描述
   - 涉及的 Action/Controller 类列表（包路径、职责）
   - 涉及的 Service 类列表（事务代理Bean、依赖DAO）
   - 涉及的数据库表列表（表名、说明）
   - 依赖的其他模块

2. **业务流程**
   - 核心业务流程图（ASCII 或 Mermaid）
   - 状态转换图
   - 流程说明

3. **接口文档**
   - 每个接口的详细说明
   - 请求参数（参数名、类型、必填、校验规则、默认值、业务含义）
   - 响应结果（成功/错误响应）
   - 处理逻辑（步骤化描述）
   - 异常处理

4. **状态码定义**
   - 所有状态码的编码和含义
   - 状态转换规则

5. **权限控制**
   - 权限检查方法
   - 权限编码定义

6. **数据模型**
   - 实体类字段定义
   - 视图对象字段定义

### 1.2 方法级参考文档标准（action-methods-reference.md）

每个方法必须包含：

1. **方法签名**
   - 完整的方法签名
   - 参数列表

2. **功能描述**
   - 方法的功能说明

3. **事务类型**
   - 是否有事务
   - 事务传播行为

4. **输入参数**
   - 参数名、类型、业务含义、校验规则

5. **返回值**
   - 返回类型
   - 返回值说明

6. **核心算法/处理逻辑**
   - 步骤化描述
   - 关键逻辑说明

7. **调用的DAO方法**
   - 列出所有调用的DAO方法

8. **调用的其他Service方法**
   - 列出所有调用的其他Service方法

9. **异常处理**
   - 异常类型、触发条件、处理方式

10. **注意事项**
    - 特殊说明、遗留问题等
