# 低代码平台用户指南

> 适用模块：PMS-springmvc（低代码可视化设计器 + 运行时渲染引擎）
> 元数据载体：`data_field_relation` 表 / `DataFieldRelation` 实体
> 安全基线：Shiro 权限标识 `lowcode:config:*`

---

## 1. 概述

低代码平台是 PMS-springmvc 提供的一套**配置驱动**的页面可视化搭建能力。平台通过维护 `data_field_relation`（数据字段关联）元数据，将"页面长什么样、有哪些字段、字段如何交互"全部以结构化数据描述，再由运行时渲染引擎根据元数据动态生成表单、列表、标签页与关联页，从而让业务人员或实施人员在**不写代码**的前提下完成页面搭建与调整。

### 1.1 核心概念

| 概念 | 说明 |
|------|------|
| **dataName** | 配置的业务实体名（如 `project`、`spare_part`），同一 `dataName` 下可挂多类页面 |
| **dataType** | 页面类型，取值：`form`（表单）/ `list`（列表）/ `tab`（标签页）/ `related-page`（关联页） |
| **field** | 字段标识，对应数据源中的字段名 |
| **type** | 字段渲染类型，如 `text`、`select`、`date` 等 |
| **status** | 配置状态：`draft`（草稿）/ `published`（已发布）/ `disabled`（已停用） |
| **version** | 配置版本号，发布时自增，便于回溯 |
| **templateId** | 模板分组，用于将一组配置归并为可复用模板 |

### 1.2 平台价值

- **配置即页面**：字段元数据 = 运行时页面，无需重启与发版。
- **所见即所得**：设计器内置预览，草稿态可实时校验效果。
- **跨环境迁移**：配置可导出为 JSON，在测试/生产间一键导入。
- **权限隔离**：基于 Shiro `lowcode:config:*` 权限控制谁能设计、谁能发布。

## 2. 新手教程

本节以"为项目实体搭建一个录入表单"为例，带你走完从进入设计器到发布的完整流程。列表、标签页、关联页的操作流程类似，差异在组件区与属性区。

### 步骤 1：进入可视化设计器

1. 使用拥有 `lowcode:config:view` 权限的账号登录系统。
2. 在导航菜单中进入「低代码平台 → 页面设计器」，打开设计器主页面 `lowcode-designer`。
3. 设计器采用**三栏布局**：
   - **左侧 组件区**：列出可拖入画布的字段类型。
   - **中间 画布区**：展示当前页面的字段排列，支持增删与排序。
   - **右侧 属性区**：编辑选中字段的全部属性。

### 步骤 2：选择页面类型

1. 画布顶部选择页面类型（`dataType`）：
   - **Form 表单**：用于数据录入与编辑。
   - **List 列表**：用于数据查询与展示。
   - **Tab 标签页**：聚合多个子页签，每个页签引用一个已有 `dataName`。
   - **Related Page 关联页**：将一页面配置为另一实体的关联详情页，配置关联键映射。
2. 填写 `dataName`（如 `project`），命名规范见 [第 5 节最佳实践](#5-最佳实践)。

### 步骤 3：添加字段

1. 从左侧组件区点击或拖拽字段类型到画布区（如 `text`、`select`）。
2. 画布区会按添加顺序展示字段，可通过拖拽调整 `sort` 顺序。
3. 每个字段会自动带出默认 `field`、`title`，可在属性区修改。

### 步骤 4：配置字段属性

1. 在画布区点选某个字段，右侧属性区加载该字段全部属性。
2. 必须配置的基础属性：`field`（字段标识）、`title`（显示标题）、`type`（渲染类型）。
3. 按需配置交互属性：`required`、`readonly`、`disabled`、`visible` 等。
4. 特殊类型需补充数据源：
   - `select`：绑定数据字典（选择字典类型）。
   - `urlSelector` / `autocomplete`：配置远程数据源 URL 及参数。
5. 属性说明详见 [第 3 节组件配置说明](#3-组件配置说明)。

### 步骤 5：预览

1. 点击画布上方「预览」按钮。
2. 预览器会向 `/lowcode/config/preview` 请求**当前草稿态**配置，并复用运行时 Vue 渲染组件渲染页面。
3. 预览不改变配置状态，可反复调整后再次预览。
4. 在预览中尝试填写/查询，确认字段交互符合预期。

### 步骤 6：保存草稿与发布

1. 点击「保存草稿」：配置以 `status=draft` 写入，不对外生效。
2. 确认无误后点击「发布」：
   - 校验字段 `type` 合法性、必填项、`dataName + dataType` 唯一性。
   - 校验通过后，当前草稿转为 `published`，**同 `dataName + dataType` 的旧发布版本自动置为 `disabled`**。
   - 版本号 `version` 自增，保留历史版本以便回溯。
3. 发布后，运行时渲染引擎即按新版本渲染对应页面。

> 状态机：`draft`（草稿） → `published`（已发布） → `disabled`（已停用）。已停用配置可再次发布恢复生效。

## 3. 组件配置说明

### 3.1 字段类型（type）

| type | 用途 | 典型场景 | 备注 |
|------|------|----------|------|
| `text` | 单行文本输入 | 名称、编码 | 默认输入框 |
| `textarea` | 多行文本输入 | 描述、备注 | 支持换行 |
| `date` | 日期选择（日） | 计划日期 | 格式 `yyyy-MM-dd` |
| `datetime` | 日期时间选择 | 创建时间 | 格式 `yyyy-MM-dd HH:mm:ss` |
| `daterange` | 日期区间选择 | 查询起止时间 | 返回起止两个值 |
| `select` | 下拉选择 | 状态、类型 | 支持绑定数据字典 |
| `urlSelector` | 远程数据选择器 | 选择关联实体 | 配置远程数据源 URL |
| `autocomplete` | 自动补全输入 | 模糊搜索实体 | 输入触发远程检索 |
| `inputs` | 多项动态输入 | 多个标签、多个值 | 动态增删行 |
| `hidden` | 隐藏字段 | 传递上下文 ID | 不在页面展示 |

### 3.2 字段属性

下表为字段通用属性（对应 `DataFieldRelation` 实体字段）。

| 属性 | 类型 | 说明 |
|------|------|------|
| `field` | String | 字段标识，对应数据源字段名，**必填** |
| `alias` | String | 字段别名，用于特殊渲染逻辑标识 |
| `name` | String | 字段名（HTML name 属性） |
| `title` | String | 字段显示标题，**必填** |
| `titleKey` | String | 标题国际化 Key（多语言场景） |
| `type` | String | 字段渲染类型，见 [3.1](#31-字段类型type)，**必填** |
| `render` | String | 自定义渲染处理（如格式化器） |
| `required` | Boolean | 是否必填，`true` 提交时校验非空 |
| `readonly` | Boolean | 是否只读，`true` 仅展示不可改 |
| `disabled` | Boolean | 是否禁用，`true` 组件灰化不可交互 |
| `cssId` | String | 组件 DOM 的 id 属性 |
| `cssClass` | String | 组件 DOM 的 class 属性 |
| `cssStyle` | String | 组件内联样式 |
| `sort` | Integer | 字段排序权重，数值越小越靠前 |
| `visible` | Boolean | 是否可见，`false` 在页面/列中隐藏 |
| `searchable` | Boolean | 是否可搜索（列表页查询条件） |
| `orderable` | Boolean | 是否可排序（列表页表头排序） |
| `extData` | String | 扩展数据（JSON 字符串），存放自定义配置 |
| `extKey` | String | 扩展数据 key |
| `extValue` | String | 扩展数据 value |
| `media` | String | 传播媒介（响应式/打印等媒介策略） |
| `clazzName` | String | 自定义渲染器类名，指向运行时渲染实现 |
| `superData` | Integer | 父类数据标识，用于关联页父子关系 |
| `isSystemField` | Boolean | 是否系统字段，`true` 不允许删除 |

> 提示：`extData` 常用于存放字典编码、远程 URL、参数模板等类型专属配置，设计器会按 `type` 解析其结构。

## 4. 模板复用

为避免重复搭建相似页面，平台支持配置的**导入导出**，实现模板沉淀与跨环境迁移。

### 4.1 导出

1. 在配置列表页勾选一个或多个 `dataName`。
2. 点击「导出」按钮，后端 `/lowcode/config/export` 接口按 `dataName` 列表导出。
3. 导出内容为 JSON，包含完整字段元数据 + 版本信息 + `dataName`/`dataType`。
4. 批量导出时，多个页面打包为 JSON 数组，便于整包迁移。

### 4.2 导入

1. 在配置列表页点击「导入」按钮，上传 JSON 文件。
2. 后端 `/lowcode/config/import` 接口执行校验：
   - **结构校验**：JSON Schema 是否合法。
   - **字段类型校验**：每个字段 `type` 是否在允许枚举内。
   - **必填校验**：`field`/`title`/`type` 等必填项是否齐全。
3. 校验通过后，配置以 `status=draft`（草稿）写入，**不会直接生效**，需人工确认后发布。
4. 校验失败时返回明细错误列表（行号 + 字段 + 原因），便于逐项修正。

### 4.3 跨环境迁移

| 场景 | 做法 |
|------|------|
| 测试 → 生产 | 在测试环境导出 JSON，上传至生产环境导入为草稿，验证后发布 |
| 生产 → 本地 | 导出生产配置到本地设计器复现问题，避免污染生产数据 |
| 模板沉淀 | 将通用页面（如基础信息表单）导出存档，新项目导入后微调 |

> 注意：导入的草稿在发布前需核对 `dataName`/`dataType` 是否与目标环境现有已发布配置冲突，避免覆盖线上页面。

## 5. 最佳实践

### 5.1 命名规范

- **dataName**：业务实体名，全小写 + 下划线，如 `project`、`spare_part`、`rma_apply`，同一实体跨页面类型保持一致。
- **dataType**：仅使用 `form`/`list`/`tab`/`related-page` 四种枚举值。
- **field**：与数据库字段名对齐，避免中文与特殊字符。
- **title**：用户可见文案，简洁明了，避免与技术字段同名。

### 5.2 状态工作流

```
draft (草稿) ──发布──▶ published (已发布) ──停用──▶ disabled (已停用)
                          ▲                                   │
                          └─────────再次发布──────────────────┘
```

- **draft**：可任意修改，不对外生效；预览仅消费草稿。
- **published**：唯一对运行时生效的状态；同 `dataName + dataType` 仅允许一个 `published`。
- **disabled**：历史版本归档，保留可回溯；需要时可重新发布恢复。

发布动作会自动将旧的 `published` 版本置为 `disabled`，无需手动停用。

### 5.3 版本管理

- 每次发布 `version` 自增，发布历史可在详情页查看。
- 回滚方案：将目标历史版本复制为新草稿 → 校验 → 发布，避免直接改动历史版本。
- 重要变更建议先在草稿态充分预览，确认无回归后再发布。

### 5.4 权限管理

低代码平台基于 Shiro 权限标识控制操作范围，建议按角色分配：

| 权限标识 | 含义 | 建议角色 |
|----------|------|----------|
| `lowcode:config:view` | 查看配置与设计器 | 实施人员、业务管理员 |
| `lowcode:config:edit` | 编辑草稿、导入导出 | 实施人员 |
| `lowcode:config:publish` | 发布与停用 | 业务管理员（审批后） |
| `lowcode:config:delete` | 删除配置 | 超级管理员（谨慎授予） |

> 发布为高风险操作，建议将 `publish` 权限与 `edit` 权限分离，由不同人员复核后发布。

## 6. 常见问题

### Q1：预览效果与正式发布页面不一致？

**排查**：
1. 检查配置 `status`：预览渲染的是**草稿（draft）**，运行时渲染的是**已发布（published）**。若草稿改了但没发布，正式页面仍是旧版本。
2. 确认是否误将草稿当作已发布：在设计器详情页查看 `status` 字段。
3. 确认浏览器缓存：强制刷新或清除缓存后重试。

### Q2：导入 JSON 失败？

**排查**：
1. **结构错误**：检查 JSON Schema 是否合法（顶层字段、数组结构、必填项缺失）。导入接口会返回明细错误，按行号修正。
2. **字段类型非法**：每个字段的 `type` 必须在允许枚举内（见 [3.1](#31-字段类型type)）。
3. **必填项缺失**：`field`/`title`/`type` 不可为空。
4. **版本不兼容**：跨大版本导出的 JSON 可能包含新字段，旧版本环境无法识别，需对齐平台版本。

### Q3：提示"权限不足 / Permission denied"？

**排查**：
1. 确认当前账号是否拥有对应 Shiro 权限：
   - 仅查看设计器：需要 `lowcode:config:view`。
   - 编辑/导入导出：需要 `lowcode:config:edit`。
   - 发布/停用：需要 `lowcode:config:publish`。
2. 联系系统管理员在角色管理中分配对应权限标识。
3. 若已分配权限仍提示拒绝，检查 Shiro 缓存是否刷新（重新登录或等待缓存过期）。

### Q4：发布时报"dataName + dataType 已存在已发布配置"？

**说明**：同 `dataName + dataType` 仅允许一个 `published` 版本。发布动作本会自动将旧版本置为 `disabled`，若仍报冲突，通常是校验逻辑检测到异常状态（如多个 published 并存），请联系管理员在数据库排查 `data_field_relation` 表 `status` 字段，修正脏数据后重试。

### Q5：标签页/关联页引用的子页面不显示？

**排查**：
1. 被引用的 `dataName` 必须处于 `published` 状态，草稿态不对外生效。
2. 检查标签页配置中子页签的 `visible` 是否为 `true`、`sort` 是否正确。
3. 关联页需正确配置关联键映射（`superData` / 关联字段），否则无法关联父实体数据。

## 附录 A：用户反馈模块

低代码平台内置**用户反馈**通道（实现在 core 模块），便于使用过程中提交问题与建议。

### A.1 反馈入口

反馈入口建议挂载到系统**顶栏菜单**（全局可见），点击弹出反馈表单，提交后由管理员在「反馈管理」列表处理。

> 部署提示：反馈的 JSP 页面（提交弹窗、管理列表）需在 PMS-struts / PMS-springmvc 的 webapp 目录中创建，并在菜单管理中配置顶栏「反馈」入口指向 `/admin/feedback`。后端接口已在 core 模块就绪，前端页面与菜单接线按实际部署环境完成。

### A.2 反馈字段

| 字段 | 说明 |
|------|------|
| `type` | 反馈类型：`bug`（缺陷）/ `suggestion`（建议）/ `question`（咨询） |
| `title` | 标题 |
| `content` | 详细内容 |
| `contact` | 联系方式 |
| `status` | 处理状态：`open`（待处理）→ `processing`（处理中）→ `resolved`（已解决）→ `closed`（已关闭） |

### A.3 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/admin/feedback/submit` | POST | 提交反馈（AJAX，返回 JSON） |
| `/admin/feedback/list` | GET | 反馈列表（管理端） |
| `/admin/feedback/{id}` | GET | 反馈详情 |
| `/admin/feedback/updateStatus` | POST | 更新反馈状态 |

建表脚本见 `core/docs/migration/add_feedback_table.sql`。
